# SBIR 計畫執行指南

> 通過審查只是開始，成功結案才是目標。

## 通過後的第一步

### 恭喜！您通過了！

收到核定通知後，接下來的流程：

```
核定通知 → 簽約準備 → 正式簽約 → 啟動會議 → 計畫執行 → 期中查核 → 期末結案
    ↓          ↓          ↓          ↓          ↓          ↓          ↓
  1週內     2-4週      當天       1個月      持續      計畫中期    計畫結束
```

### 簽約準備清單

| 項目 | 說明 | 注意事項 |
|------|------|----------|
| 公司大小章 | 負責人章 + 公司章 | 需與登記一致 |
| 公司登記證明 | 變更登記表或設立登記表 | 最新版本 |
| 銀行帳戶 | 專戶或公司帳戶 | 需為公司戶 |
| 計畫負責人印章 | 計畫主持人個人章 | 與申請書一致 |
| 研發人員名冊 | 參與計畫人員清單 | 需含學經歷 |

### 專戶開立

**是否需要專戶？**
- Phase 1（< 150 萬）：通常不需要，可用公司帳戶
- Phase 2（> 500 萬）：建議開立專戶便於管理

**專戶優點**：
- 收支清楚
- 便於查核
- 避免混淆

## 計畫啟動

### 啟動會議

通常在簽約後 1 個月內舉行：

**會議內容**：
1. 計畫內容說明
2. 查核點確認
3. 經費使用規範
4. 行政作業說明
5. Q&A

**準備事項**：
- 簡報資料（計畫概述）
- 執行時程表
- 團隊成員名單
- 問題清單

## 進度管理

### 每月工作

| 項目 | 頻率 | 負責人 |
|------|------|--------|
| 進度追蹤 | 每週 | 計畫主持人 |
| 經費動支 | 每月 | 財務負責人 |
| 文件歸檔 | 即時 | 行政負責人 |
| 團隊會議 | 每週/雙週 | 計畫主持人 |

### 進度追蹤表範例

```
┌─────────────────────────────────────────────────────────┐
│                    計畫進度追蹤表                       │
├─────────┬──────────┬──────────┬──────────┬─────────────┤
│  工作項 │ 預定完成 │ 實際完成 │   狀態   │    備註     │
├─────────┼──────────┼──────────┼──────────┼─────────────┤
│ 需求分析│  M1      │  M1      │  ✓完成   │             │
│ 系統設計│  M2      │  M2      │  ✓完成   │             │
│ 模組開發│  M4      │  進行中  │  ⏳執行中│ 完成 70%    │
│ 整合測試│  M5      │    -     │  ○待執行 │             │
│ 實場驗證│  M6      │    -     │  ○待執行 │             │
└─────────┴──────────┴──────────┴──────────┴─────────────┘
```

### 進度落後處理

**發現落後時**：
1. 分析原因（技術困難？資源不足？外部因素？）
2. 評估影響（可否追回？需要調整？）
3. 擬定對策（加人力？調整方法？）
4. 必要時申請變更

**變更申請時機**：
- 計畫內容重大調整
- 執行期程需延長
- 經費需重新分配（跨項超過 20%）
- 主要人員異動

## 經費管理

### 經費類別

| 類別 | 內容 | 注意事項 |
|------|------|----------|
| 人事費 | 研發人員薪資 | 需有勞保證明 |
| 消耗性器材費 | 材料、耗材 | 需有發票 |
| 設備使用費 | 儀器租用 | 需有合約 |
| 技術移轉費 | 技術授權 | 需有合約 |
| 委託研究費 | 外包研發 | 需有合約+成果 |
| 差旅費 | 出差交通住宿 | 需有單據 |
| 管理費 | 間接成本 | 有上限比例 |

### 經費動支流程

```
需求提出 → 事前申請 → 核准 → 採購/支出 → 單據核銷 → 歸檔
```

### 經費使用注意事項

**絕對禁止**：
- ❌ 補助款用於非計畫用途
- ❌ 虛報費用
- ❌ 非研發相關支出

**需要留意**：
- ⚠️ 人事費通常有上限比例
- ⚠️ 設備購置可能需事先報備
- ⚠️ 跨類別流用有比例限制

### 單據管理

**必備單據**：
- 發票或收據
- 採購證明（比價單、合約）
- 驗收證明
- 付款證明（匯款單或簽收單）

**單據整理要點**：
1. 依類別分類
2. 依時間排序
3. 黏貼整齊
4. 建立清冊

## 查核點準備

### 查核類型

| 類型 | 時間 | 內容 |
|------|------|------|
| 季報 | 每季 | 書面報告 |
| 期中查核 | 計畫中期 | 書面 + 簡報 |
| 期末結案 | 計畫結束 | 完整報告 + 簡報 |

### 季報撰寫

**報告結構**：
1. 本季工作摘要
2. 計畫進度（vs 預定）
3. 經費執行情形
4. 遭遇問題與解決方案
5. 下季工作重點

**報告技巧**：
- 用數據說明進度
- 誠實報告問題
- 提出具體解決方案
- 附上佐證資料

### 期中查核

**準備事項**：
1. 期中報告書
2. 簡報資料（15-20 分鐘）
3. 成果展示（雛型、數據）
4. 經費執行報告

**簡報技巧**：
- 聚焦重點成果
- 展示具體進展
- 說明技術突破
- 面對問題不迴避

**常見問題**：
- 「目前進度與預期的差異？」
- 「遇到什麼技術困難？如何解決？」
- 「經費執行率為何偏低/高？」
- 「下階段重點工作是什麼？」

## 常見問題處理

### 問題一：進度落後

**症狀**：實際進度明顯落後預定進度

**處理方式**：
1. 分析落後原因
2. 制定追趕計畫
3. 若無法追回，提早申請展延
4. 在報告中誠實說明

**申請展延條件**：
- 非歸責於執行單位
- 有合理說明
- 展延後可完成目標

### 問題二：經費不足

**症狀**：自籌款不足以支應計畫

**處理方式**：
1. 檢視經費配置
2. 調整優先順序
3. 申請經費流用
4. 必要時縮減計畫範圍

### 問題三：人員異動

**症狀**：主要研發人員離職

**處理方式**：
1. 盡快補充人力
2. 確保技術交接
3. 向計畫辦公室報備
4. 更新人員名冊

### 問題四：技術困難

**症狀**：技術瓶頸無法突破

**處理方式**：
1. 尋求外部專家協助
2. 調整技術路線
3. 必要時修正計畫目標
4. 記錄探索過程（也是成果）

## 結案準備

### 結案報告結構

1. **摘要**
   - 計畫目標
   - 主要成果
   - 效益評估

2. **執行內容**
   - 各階段工作說明
   - 技術成果
   - 突破與創新

3. **成果展示**
   - 產品/技術展示
   - 量化成果
   - 專利/論文

4. **效益評估**
   - 技術效益
   - 經濟效益
   - 社會效益

5. **後續規劃**
   - 商品化計畫
   - 市場推廣
   - 持續研發

### 結案審查

**審查方式**：
- 書面審查
- 現場簡報（通常 20-30 分鐘）
- 成果展示
- Q&A

**審查重點**：
- 目標達成度
- 成果品質
- 經費運用合理性
- 後續發展潛力

### 常見缺失

| 缺失 | 說明 | 如何避免 |
|------|------|----------|
| 目標未達成 | 查核指標未完成 | 定期追蹤、及早反應 |
| 成果不明確 | 無法具體說明成果 | 量化呈現、保留記錄 |
| 單據不全 | 經費核銷有問題 | 即時整理、定期檢查 |
| 報告空泛 | 內容缺乏具體數據 | 用數據說話 |
| 智財未處理 | 未辦理專利/技轉 | 提早規劃 |

## 智財成果申報

### 智財類型

| 類型 | 說明 | 是否需申報 |
|------|------|------------|
| 發明專利 | 新技術、新方法 | ✓ 必須 |
| 新型專利 | 物品形狀構造 | ✓ 必須 |
| 設計專利 | 外觀設計 | ✓ 必須 |
| 營業秘密 | 技術 know-how | 建議登記 |
| 著作權 | 軟體、文件 | ✓ 必須 |

### 專利申請建議

**申請時機**：
- 計畫執行中即可申請
- 不需等結案
- 2025 年起有專利費用補助

**申請策略**：
1. 辨識可專利技術
2. 進行專利檢索
3. 提早佈局
4. 考慮國際專利

## 計畫結案後

### 成果追蹤

結案後通常需配合：
- 年度成果追蹤問卷
- 技轉或商品化追蹤
- 績效統計

### 後續發展

**成功完成後的選項**：

1. **申請 Phase 2**
   - 擴大研發規模
   - 最高補助 1,200 萬

2. **申請 Phase 2+**
   - 加值應用
   - 商品化推動

3. **技術移轉**
   - 授權給其他企業
   - 創造營收

4. **商品化**
   - 直接推向市場
   - 建立商業模式
