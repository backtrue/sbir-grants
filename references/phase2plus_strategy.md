# Phase 2+ 加值應用撰寫策略

## 概述

Phase 2+（加值應用）是 SBIR 的商業化階段，目標是將 Phase 2 完成的研發成果推向市場，進行商業化與規模化。

| 項目 | 內容 |
|------|------|
| 補助上限 | 600 萬元（2026年起） |
| 期程 | 依計畫而定 |
| 補助比例 | 最高 50% |
| 前提條件 | 通常需完成 Phase 2 |

---

## 核心目標

Phase 2+ 要回答的核心問題：

> **「你的產品/服務如何進入市場並創造營收？」**

具體而言：
1. 如何從產品雛形到量產/上線？
2. 如何建立銷售通路？
3. 如何擴大市場規模？
4. 如何實現財務自立？

---

## 適用情境

Phase 2+ 適合以下情況：
- ✅ 已完成 Phase 2，有初步產品/服務
- ✅ 已有客戶驗證或早期採用者
- ✅ 需要資金支持商業化拓展
- ✅ 有明確的市場進入策略

---

## 計畫書重點

### 與 Phase 2 的差異

| 面向 | Phase 2 | Phase 2+ |
|------|---------|----------|
| 重點 | 研發、驗證 | 商業化、規模化 |
| 產出 | 產品雛形、POC | 量產、市場導入 |
| 內容 | 技術開發為主 | 市場拓展為主 |
| 經費用途 | 研發人力、設備 | 行銷、通路、量產 |

### 計畫書架構

```
1. 公司概況與 Phase 2 成果 (2-3 頁)
2. 產品/服務說明 (2-3 頁)
3. 商業化策略 (5-8 頁)
4. 市場拓展計畫 (3-5 頁)
5. 營運與財務規劃 (3-5 頁)
6. 團隊與資源 (2-3 頁)
7. 經費編列 (2 頁)
8. 預期效益 (2 頁)
```

---

## 關鍵章節撰寫

### 1. Phase 2 成果展示

**必須說明**：
- Phase 2 完成了什麼
- 技術驗證結果
- 客戶驗證成果
- 目前產品/服務狀態

**證據展示**：
- 技術指標達成數據
- 客戶/用戶回饋
- POC/試用結果

### 2. 商業化策略

**必須說明**：
- 產品/服務如何量產或上線
- 定價策略
- 銷售通路
- 品牌與行銷策略
- 售後服務規劃

**商業化路徑圖**：

```
Phase 2 成果 → 量產/上線準備 → 首批客戶導入 → 市場擴展 → 規模化
```

### 3. 市場拓展計畫

**內容**：
- 目標市場優先順序
- 客戶開發策略
- 通路合作計畫
- 行銷推廣方案
- 年度銷售目標

**表格建議**：

| 年度 | 目標客戶數 | 營收目標 | 主要行動 |
|------|------------|----------|----------|
| Y1 | 10家 | 2,000萬 | 種子客戶、案例建立 |
| Y2 | 30家 | 5,000萬 | 通路擴展、品牌行銷 |
| Y3 | 80家 | 1億 | 規模化、新市場 |

### 4. 財務規劃

**內容**：
- 成本結構分析
- 定價與毛利率
- 損益預估（3-5年）
- 資金需求與來源
- 損益平衡點分析

---

## 審查重點

Phase 2+ 審查特別重視：

1. **Phase 2 成果完整性**
   - 技術是否已驗證
   - 產品/服務是否已就緒

2. **商業化可行性**
   - 市場進入策略是否合理
   - 銷售通路是否可行

3. **財務規劃合理性**
   - 營收預估是否有根據
   - 成本控制是否合理
   - 損益平衡時間是否合理

4. **團隊商業能力**
   - 除技術外是否有商業人才
   - 是否有銷售/行銷經驗

---

## 經費編列注意

Phase 2+ 經費使用與 Phase 2 不同：

| 類別 | Phase 2 重點 | Phase 2+ 重點 |
|------|--------------|---------------|
| 人事費 | 研發人員 | 業務/行銷人員 |
| 設備費 | 研發設備 | 生產/量產設備 |
| 其他費 | 技術開發 | 行銷推廣、展會 |

---

## 常見問題

### 1. Phase 2 成果不足

**問題**：產品/服務尚未真正完成
**對策**：確保 Phase 2 確實完成再申請

### 2. 商業化策略模糊

**問題**：只說要賣，沒說怎麼賣
**對策**：具體說明通路、定價、行銷策略

### 3. 財務預估過於樂觀

**問題**：營收成長預估不切實際
**對策**：使用保守假設，說明計算基礎

### 4. 缺乏商業人才

**問題**：團隊全是技術背景
**對策**：補充商業顧問或業務夥伴
