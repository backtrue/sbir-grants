# SBIR 簡報技巧指南

> 簡報是計畫書的「現場考試」，15 分鐘決定你的成敗。

## 簡報的重要性

### 審查流程中的簡報

```
計畫送件 → 書面初審 → 簡報複審 → 核定
                         ↑
                    決勝點！
```

**簡報佔比**：
- 書面審查：60%
- 簡報審查：40%

很多書審分數接近的案子，最終由 **簡報表現** 決定勝負。

---

## 簡報結構設計

### 標準結構（15 分鐘）

| 段落 | 時間 | 內容 |
|------|------|------|
| 開場 | 1 分鐘 | 吸引注意、自我介紹 |
| 問題 | 2 分鐘 | 痛點、市場需求 |
| 解方 | 4 分鐘 | 創新技術、核心價值 |
| 驗證 | 2 分鐘 | 前期成果、可行性 |
| 市場 | 2 分鐘 | TAM/SAM/SOM、商業模式 |
| 團隊 | 2 分鐘 | 團隊能力、為什麼是我們 |
| 總結 | 2 分鐘 | 經費、時程、預期效益 |

### 投影片建議張數

| 計畫類型 | 簡報時間 | 投影片張數 |
|----------|----------|------------|
| Phase 1 | 10-15 分鐘 | 12-18 張 |
| Phase 2 | 15-20 分鐘 | 18-25 張 |

**原則**：每張投影片約 1 分鐘

## 開場技巧

### 黃金 30 秒

**目標**：在前 30 秒內讓評審知道「這是什麼、為什麼重要」

#### 開場範例

**❌ 無聊開場**
> 「各位評審委員好，我是 XX 公司的 OOO，今天要報告的是 SBIR 計畫，計畫名稱是『智慧化生產管理系統開發計畫』...」

**✅ 吸睛開場**
> 「各位評審好！製造業每年因為排程混亂導致的訂單延遲，造成超過 500 億損失。今天我們要展示的這套 AI 排程系統，已經在 3 家工廠實測，讓準時交付率從 60% 提升到 95%。我是 XX 公司創辦人 OOO...」

### 開場的 4 種策略

1. **數據開場**
   > 「台灣中小製造業有 87% 仍在使用 Excel 做生產管理...」

2. **故事開場**
   > 「去年，一家百年老店因為一場設備故障差點倒閉...」

3. **問題開場**
   > 「請問在座的評審，有沒有等過外送超過一小時？」

4. **願景開場**
   > 「想像一下，三年後每一台工具機都能在故障前自己發出警報...」

## 內容呈現技巧

### 視覺設計原則

#### 字體與文字
- 標題：**36-44 pt**
- 內文：**24-28 pt**
- 每張投影片文字不超過 **7 行**
- 關鍵字用**粗體**或顏色突顯

#### 圖表使用
- 優先用圖代替文字
- 每張投影片最多 **1-2 張圖**
- 圖表要有標題和數據標籤

#### 顏色配置
- 主色調不超過 **3 種**
- 對比色突顯重點
- 背景簡潔（白底或淺色）

### 投影片範例結構

```
┌────────────────────────────────────────┐
│  標題（簡潔有力）                       │
├────────────────────────────────────────┤
│                                        │
│   重點 1    重點 2    重點 3           │
│     ●         ●         ●              │
│    文字      文字      文字            │
│                                        │
│         [核心圖表或示意圖]              │
│                                        │
│  ➜ 關鍵訊息或結論                      │
│                                        │
└────────────────────────────────────────┘
```

## 技術呈現技巧

### 讓非專家也能理解

**原則**：評審不一定是你領域的專家

**技巧**：
1. **用比喻**：「這就像幫機器裝上聽診器」
2. **用對比**：「傳統方法 vs 我們的方法」
3. **視覺化**：用圖說明架構和流程
4. **量化**：用數字說明效果

### 技術簡報常見錯誤

| 錯誤 | 問題 | 解法 |
|------|------|------|
| 術語過多 | 評審聽不懂 | 用白話解釋 |
| 細節過多 | 失焦 | 聚焦核心創新 |
| 沒有數據 | 不具說服力 | 量化效果 |
| 缺少示意圖 | 抽象難懂 | 視覺呈現 |

## Q&A 應對策略

### 常見問題類型

#### 1. 技術可行性
> 「這個技術準確率怎麼達到的？」

**回答技巧**：
- 說明技術原理
- 提供實驗數據
- 承認限制並說明解決方案

#### 2. 市場競爭
> 「市場上已經有類似產品，你們的差異是什麼？」

**回答技巧**：
- 具體說明差異點
- 用數據對比
- 強調你的獨特優勢

#### 3. 商業模式
> 「你們打算怎麼賺錢？」

**回答技巧**：
- 清楚說明收費模式
- 說明目標客戶
- 提供營收預估

#### 4. 團隊能力
> 「你們團隊有相關經驗嗎？」

**回答技巧**：
- 說明團隊成員背景
- 展示過往成果
- 強調互補性

#### 5. 經費使用
> 「人事費佔比這麼高，合理嗎？」

**回答技巧**：
- 解釋編列依據
- 說明工作內容對應
- 展示效益

### Q&A 黃金法則

1. **聽完再答**：不要急著搶答
2. **正面回應**：不要迴避問題
3. **簡潔有力**：不要長篇大論
4. **用數據**：有數據就用數據
5. **誠實**：不知道就說「我們會進一步研究」

## 時間掌控

### 練習技巧

1. **計時練習**：至少練習 5 次以上
2. **錄影回看**：找出問題點
3. **模擬 Q&A**：準備 20 個可能問題
4. **同事演練**：請不懂的人當評審

### 時間分配參考

**15 分鐘簡報時間分配**：

```
0:00  ─┬─  開場
       │
1:00  ─┼─  問題陳述
       │
3:00  ─┼─  解決方案（核心！）
       │
7:00  ─┼─  前期成果/驗證
       │
9:00  ─┼─  市場分析
       │
11:00 ─┼─  團隊介紹
       │
13:00 ─┼─  經費/時程
       │
14:00 ─┼─  結論/願景
       │
15:00 ─┴─  謝謝，開始 Q&A
```

## 臨場技巧

### 緊張處理

**緊張是正常的！**

**緩解方法**：
1. **提前到場**：熟悉環境
2. **深呼吸**：上台前 3 次深呼吸
3. **喝水**：緩解口乾
4. **看友善的臉**：找一個看起來友善的評審看著

### 肢體語言

| 要做 | 避免 |
|------|------|
| 眼神接觸 | 一直盯螢幕或筆記 |
| 自然手勢 | 雙手插口袋或背後 |
| 站穩 | 不斷踱步 |
| 微笑 | 面無表情 |

### 聲音表達

- **語速**：比平常慢一些
- **音量**：確保後排能聽到
- **停頓**：重要觀點後暫停一秒
- **變化**：避免單調

## 評審觀察重點

### 評審在看什麼？

| 面向 | 評審會想 |
|------|----------|
| 專業度 | 「這個人/團隊懂不懂自己在做什麼？」 |
| 邏輯性 | 「問題→解方→效益的邏輯通嗎？」 |
| 可行性 | 「這個計畫真的做得出來嗎？」 |
| 市場性 | 「做出來有人會買嗎？」 |
| 信心 | 「他們有信心能成功嗎？」 |

### 加分行為

- ✅ 展示實際樣品或 Demo
- ✅ 有客戶見證或意向書
- ✅ 數據清楚有來源
- ✅ 回答問題有條理
- ✅ 坦承限制並說明對策

### 扣分行為

- ❌ 照稿念
- ❌ 閃躲問題
- ❌ 時間超過太多
- ❌ 投影片太雜亂
- ❌ 無法回答基本問題
