# 經費編列說明與委外合理性指南

*基於真實 SBIR 審查案例*

## 📊 經費編列概覽

### 真實案例統計

| 公司 | 總經費 | 人事費比例 | 委外費用 | 特點 |
|------|--------|-----------|----------|------|
| B公司 | 300萬 | ~40% | 有 | 標準配置 |
| F公司 | 1000萬 | ~40% | 有 | 接近上限 |
| A公司 | - | ~30% | 高 | 委外為主 |

**關鍵發現**：
- ✅ 人事費通常佔 30-50%
- ✅ 委外是常見做法，但**必須說明合理性**
- ✅ 經費規模差異大（300萬 - 1000萬）

---

## 💰 經費項目說明

### 1. 人事費

#### 編列原則

**上限**：原則上不超過總經費 50%

**包含項目**：
- 研發人員薪資
- 勞健保
- 勞退提撥

#### 真實案例說明方式

```
人事費：480 萬元（40%）

人力配置：
1. 計畫主持人（1人）
   - 投入時數：每月 80 小時
   - 月薪資：12 萬元
   - 計畫期間：24 個月
   - 小計：12萬 × 24月 × 33% = 95萬

2. 研發工程師（2人）
   - 投入時數：每月 160 小時（全職）
   - 月薪資：8 萬元/人
   - 計畫期間：24 個月
   - 小計：8萬 × 2人 × 24月 = 384萬

3. 勞健保及勞退：約 1萬/人/月
   - 小計：1萬 × 3人 × 24月 = 72萬

總計：551萬（調整後 480萬）
```

**關鍵要素**：
- ✅ 列出每個人的投入時數
- ✅ 說明薪資水準
- ✅ 計算方式透明
- ✅ 包含勞健保

---

### 2. 消耗性器材費

#### 包含項目

- 材料、零組件
- 試劑、耗材
- 軟體授權費（非永久）
- 雲端服務費

#### 真實案例：軟體開發類

```
消耗性器材費：150 萬元（15%）

項目明細：
1. 雲端主機費用
   - AWS EC2 + S3 服務
   - 預估：5萬/月 × 24月 = 120萬

2. API 服務費用
   - OpenAI API
   - 預估：1萬/月 × 24月 = 24萬

3. 開發工具授權
   - IDE、測試工具年度授權
   - 預估：6萬

總計：150萬
```

**關鍵要素**：
- ✅ 說明用途
- ✅ 提供估算依據
- ✅ 月費 × 月數清楚列出

---

### 3. 委託研究費

#### ⚠️ 最容易被質疑的項目

#### 常見審查意見

> Q: 計畫簡報自陳「初期系統開發以委外方式推動」，請敘明委外開發的必要性和合理性，尤其當公司自陳技術為核心能量所在

#### 真實案例回覆（A公司）

```
委託研究費：240 萬元（20%）

委外項目與必要性說明：

1. 人力現況
   - 核心團隊：2 名全職創辦成員
   - 專長：AI 模型開發、商務營運
   - 限制：日常營運已佔用大部分時間

2. 工作分類

   【核心工作 - 自行開發】
   ◼ AI 模型訓練與優化
   ◼ 核心演算法開發
   ◼ 服務架構設計
   → 由本公司團隊負責，確保技術掌控

   【非核心工作 - 委外協助】
   ◼ UI/UX 介面設計：60萬
     理由：非核心專長，委外效率更高
   
   ◼ 系統整合測試：80萬
     理由：需要專業測試團隊，確保品質
   
   ◼ 功能開發協助：100萬
     理由：加速開發進程，縮短上市時間

3. 委外合作模式
   - 採專案委託方式
   - 成果歸本公司所有
   - 本公司保有完整技術掌控權
   - 委外廠商：XX 資訊公司（合作 3 年經驗）

4. 技術能力內化計畫
   - 計畫期間將增聘 1 名工程師
   - 負責系統維護與後續開發
   - 確保技術能力內化，不依賴委外

結論：
委外僅針對非核心、可替代性高的工作項目，
核心技術（AI 模型、演算法）仍由本公司團隊掌握。
```

**關鍵要素**：
1. ✅ **明確區分核心/非核心**
2. ✅ **說明人力限制**
3. ✅ **強調成果歸屬**
4. ✅ **提出能力內化計畫**

---

### 4. 設備費

#### 編列原則

**Phase 1 限制**：盡量避免大型設備採購

**建議做法**：
- 優先考慮租用
- 說明必要性
- 提供採購規劃

#### 真實案例說明

```
設備費：150 萬元（15%）

項目明細：

1. 高效能工作站（2台）
   - 規格：Intel i9, 64GB RAM, RTX 4090
   - 用途：AI 模型訓練
   - 必要性：現有設備效能不足，訓練時間過長
   - 單價：60萬/台
   - 小計：120萬

2. 測試設備租用
   - 項目：專業測試儀器
   - 用途：產品驗證
   - 租用期間：12 個月
   - 小計：30萬

總計：150萬

說明：
- 工作站為必要設備，將持續使用於後續研發
- 測試設備採租用方式，避免閒置浪費
```

**關鍵要素**：
- ✅ 說明規格和用途
- ✅ 解釋必要性
- ✅ 租用 vs 購買的考量

---

### 5. 其他費用

#### 包含項目

- 差旅費
- 專利申請費
- 管理費
- 雜支

#### 編列上限

**管理費**：不超過總經費 10%

#### 真實案例

```
其他費用：180 萬元（15%）

項目明細：

1. 差旅費：36萬
   - 客戶訪談（北中南）：20萬
   - 技術交流（國內研討會）：10萬
   - 供應商拜訪：6萬

2. 專利申請費：60萬
   - 國內發明專利：15萬 × 2件 = 30萬
   - 國際專利（PCT）：30萬 × 1件 = 30萬

3. 管理費：84萬（7%）
   - 行政人力、會計、法務等

總計：180萬
```

**關鍵要素**：
- ✅ 差旅費要說明目的地和目的
- ✅ 專利費要列明件數
- ✅ 管理費不超過 10%

## 📋 經費編列檢核清單

### 基本檢核

- [ ] 總經費符合 Phase 限制（Phase 1: 150萬, Phase 2: 1,200萬）
- [ ] 補助款不超過總經費 50%
- [ ] 人事費不超過 50%
- [ ] 管理費不超過 10%

### 說明完整性

- [ ] 每個項目都有用途說明
- [ ] 委外項目有必要性說明
- [ ] 設備費有規格和必要性
- [ ] 差旅費有目的地和目的

### 合理性

- [ ] 薪資水準符合市場行情
- [ ] 委外費用有報價依據
- [ ] 設備規格符合需求（不過度）
- [ ] 各項目比例合理

## 💡 經費編列策略

### 策略 1：突顯 SBIR 補助的必要性

**在劣勢中連結經費需求**：

```
SWOT 劣勢：
資金規模較小，相較於大型競爭對手，
本公司資金有限，難以同時投入多項研發。

對策：
透過 SBIR 補助，彌補資金缺口，
讓本公司能專注投入核心技術開發，
加速產品上市時程。
```

### 策略 2：經費與目標連結

```
計畫目標：準確率從 85% 提升至 95%

對應經費：
- 人事費（AI 工程師）：負責演算法優化
- 雲端費用：增加訓練數據量（1萬 → 10萬筆）
- 委外測試：專業驗證，確保準確率達標
```

### 策略 3：自籌款來源說明

```
經費來源：

補助款：600 萬（50%）
自籌款：600 萬（50%）
  - 公司自有資金：400 萬
  - 銀行貸款：200 萬（已洽談中）

說明：
本公司對本計畫高度重視，願意投入相當資源，
SBIR 補助將有效降低財務壓力，加速研發進程。
```

## ⚠️ 常見錯誤與改進

### 錯誤 1：委外比例過高但沒說明

❌ 委外費用 500萬（50%），沒有說明

✅ 委外費用 200萬（20%），詳細說明：
   - 核心技術自行開發
   - 委外僅限非核心項目
   - 提出能力內化計畫

### 錯誤 2：人事費計算不清楚

❌ 人事費：400萬

✅ 人事費：400萬
   - 計畫主持人：投入 33%，月薪 12萬
   - 工程師 2 人：全職，月薪 8萬
   - 計算方式：[詳細列出]

### 錯誤 3：設備費沒說明必要性

❌ 購買高效能電腦：100萬

✅ 購買高效能工作站：100萬
   - 現況：現有設備訓練時間需 7 天
   - 需求：新設備可縮短至 1 天
   - 必要性：加速研發進程，提升效率

### 錯誤 4：經費與計畫目標脫節

❌ 經費項目與計畫內容對不上

✅ 每個經費項目都能對應到：
   - 工作項目
   - 查核點
   - 預期成果
