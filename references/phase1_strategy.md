# Phase 1 先期研究撰寫策略

## 概述

Phase 1（先期規劃/先期研究）是 SBIR 的入門階段，目標是驗證創新構想的可行性。相較於 Phase 2，Phase 1 的門檻較低、金額較小，但也是建立信任與累積實績的重要起點。

| 項目 | 內容 |
|------|------|
| 補助上限 | 150 萬元（2026年起） |
| 期程 | 6 個月 |
| 補助比例 | 最高 50% |
| 審查方式 | 簡報審查 |

---

## 核心目標

Phase 1 要回答的核心問題：

> **「你的創新構想可行嗎？」**

具體而言：
1. 技術上可行嗎？
2. 市場上有需求嗎？
3. 你有能力做到嗎？

---

## 簡報架構建議

Phase 1 通常以「簡報審查」為主，建議架構：

### 1. 公司簡介 (2-3 頁)

**目的**：建立信任，展現執行能力

**內容**：
- 公司基本資料（成立時間、資本額、員工數）
- 核心業務與產品
- 團隊核心能力
- 相關研發經驗與實績
- 過去政府計畫執行情況（若有）

**撰寫重點**：
- 聚焦與本計畫相關的能力
- 用數據說話（營收成長、客戶數等）
- 簡潔有力，不要流水帳

### 2. 問題陳述 (2-3 頁)

**目的**：證明這個問題值得解決

**內容**：
- 目標客戶是誰？
- 他們面臨什麼問題/痛點？
- 這個問題有多嚴重/普遍？
- 現有解決方案的不足是什麼？

**撰寫重點**：
```
問題要具體、可量化
❌ 「企業管理很困難」
✅ 「中小企業平均花費 40% 時間在人工對帳，錯誤率高達 15%」
```

**加分項目**：
- 引用市場調查數據
- 提供客戶訪談佐證
- 有潛在客戶的初步需求確認

### 3. 創新構想 (3-4 頁)

**目的**：展示你的解決方案有何創新

**內容**：
- 你的解決方案是什麼？
- 創新點在哪裡？（技術創新/服務創新/商業模式創新）
- 與現有方案相比有何優勢？
- 預期能帶來什麼效益？

**撰寫重點**：
```
創新點要明確、可比較
❌ 「我們的系統很創新」
✅ 「導入 AI 預測演算法，相較傳統方法可減少 60% 預測誤差」
```

**結構建議**：
1. 方案概述（一句話說明）
2. 技術/服務架構圖
3. 創新點說明（用表格對比現有方案）
4. 預期效益（量化）

### 4. 可行性評估規劃 (2-3 頁)

**目的**：說明如何驗證這個構想可行

**內容**：
- 你要驗證什麼？（具體的技術/市場假設）
- 如何驗證？（研究方法、測試計畫）
- 成功的標準是什麼？
- 6 個月內的具體工作項目

**撰寫重點**：
- 驗證目標要明確可量化
- 方法要合理可行
- 時程要實際（6 個月）
- 要有清楚的里程碑

**示例**：

| 驗證項目 | 方法 | 成功標準 |
|----------|------|----------|
| 演算法準確度 | 使用歷史數據測試 | 準確率 ≥ 85% |
| 客戶接受度 | 5 家潛在客戶訪談 | ≥ 3 家表達採購意願 |
| 技術可行性 | 開發 PoC 原型 | 能處理 1000 筆/秒 |

### 5. 市場初步分析 (1-2 頁)

**目的**：證明市場有足夠吸引力

**內容**：
- 目標市場範圍
- 市場規模估計
- 主要競爭者
- 你的競爭優勢

**撰寫重點**：
- 使用 TAM/SAM/SOM 框架
- 引用可信的市場數據來源
- 不需要太深入，但要合理

### 6. 團隊介紹 (1 頁)

**目的**：證明團隊有能力執行

**內容**：
- 計畫主持人資歷
- 核心團隊成員
- 各自負責的工作

**撰寫重點**：
- 聚焦與計畫相關的經歷
- 說明團隊互補性
- 若有外部顧問或合作夥伴也可列入

### 7. 時程與經費 (1 頁)

**目的**：展示計畫規劃的合理性

**內容**：
- 6 個月甘特圖
- 各階段主要產出
- 經費分配概要

**撰寫重點**：
- 時程要合理（6 個月內）
- 產出要明確
- 經費分配要符合規範

## 簡報審查技巧

### 時間掌控

- 簡報時間：15-20 分鐘
- 問答時間：10-15 分鐘
- 建議準備 12-15 張投影片

### 簡報呈現

**Do ✅**
- 使用大字體（至少 24pt）
- 多用圖表少用文字
- 關鍵數據要突出
- 每頁一個重點
- 準備好補充資料

**Don't ❌**
- 整頁都是文字
- 字太小看不清
- 動畫太花俏
- 內容太多講不完
- 沒有練習

### 問答應對

**常見問題類型**：
1. 技術細節：「你的演算法如何處理 XXX 情況？」
2. 市場驗證：「你有沒有實際客戶需求的驗證？」
3. 競爭對手：「跟 OOO 公司相比，你的優勢是什麼？」
4. 團隊能力：「你們之前有做過類似的案子嗎？」
5. 風險對策：「如果 XXX 問題發生，你如何處理？」

**應對原則**：
- 誠實回答，不要編造
- 聽完問題再回答
- 回答要簡潔精準
- 不確定的可以說「這是我們要在 Phase 1 驗證的」
- 適當承認不足，說明如何克服

## 經費編列建議

### Phase 1 經費結構

| 類別 | 建議比例 | 說明 |
|------|----------|------|
| 人事費 | 40-50% | 計畫相關人員薪資 |
| 耗材費 | 20-30% | 軟體、材料等 |
| 委託研究費 | 10-20% | 外包項目 |
| 差旅費 | 5-10% | 客戶訪談、展會 |
| 其他 | 5-10% | 文獻、雜支 |

### 編列原則

1. **合理性**：每項費用都要能解釋
2. **必要性**：與計畫工作直接相關
3. **經濟性**：不要編過高價格

### 自籌款說明

- 補助比例最高 50%，需自籌 50%
- 說明自籌款來源（公司自有資金、其他投資等）
- 展現執行決心

## 審查評分重點

### 評分維度（參考）

| 維度 | 權重 | 說明 |
|------|------|------|
| 創新性 | 25% | 構想的新穎程度 |
| 可行性 | 30% | 技術與執行的可行性 |
| 市場潛力 | 20% | 市場規模與需求 |
| 團隊能力 | 15% | 執行團隊的能力 |
| 計畫規劃 | 10% | 時程與經費的合理性 |

### 通過關鍵

1. **問題要真實**：確實存在且值得解決
2. **創新要明確**：清楚說明新在哪裡
3. **驗證要具體**：知道如何證明可行
4. **團隊要相關**：有能力執行這個計畫
5. **規劃要合理**：時程與經費實際可行

## 常見失敗原因

### 1. 創新性不足

**問題**：
- 只是現有技術的組合
- 與競品差異不明顯
- 沒有說明創新在哪

**對策**：
- 明確列出創新點
- 用比較表格凸顯差異
- 量化預期效益

### 2. 問題定義模糊

**問題**：
- 問題太抽象或太廣泛
- 沒有具體的客戶痛點
- 缺乏數據佐證

**對策**：
- 聚焦特定客戶群
- 用數據描述問題嚴重程度
- 提供客戶訪談佐證

### 3. 可行性說明薄弱

**問題**：
- 驗證方法不清楚
- 成功標準模糊
- 時程不合理

**對策**：
- 具體說明如何驗證
- 設定可量化的成功標準
- 合理安排 6 個月時程

### 4. 簡報表現不佳

**問題**：
- 投影片太多/太少
- 說明不清楚
- 問答應對不當

**對策**：
- 多次練習
- 控制時間
- 準備 Q&A

## Phase 1 成功後的下一步

Phase 1 完成後，若可行性驗證成功：

1. **整理成果**
   - 技術驗證報告
   - 市場調查結果
   - PoC 成果展示

2. **規劃 Phase 2**
   - 詳細的研發計畫
   - 完整的商業計畫
   - 更大的經費需求

3. **申請銜接**
   - Phase 1 成果可作為 Phase 2 的基礎
   - 展現執行能力與計畫延續性

## 檢核清單

### 申請前檢核

- [ ] 確認公司符合中小企業資格
- [ ] 確認無排除條件
- [ ] 準備公司登記文件
- [ ] 準備最近年度財務報表

### 簡報內容檢核

- [ ] 公司簡介清楚呈現相關能力
- [ ] 問題陳述具體且有數據佐證
- [ ] 創新構想明確說明創新點
- [ ] 可行性評估有具體驗證方法
- [ ] 市場分析合理有據
- [ ] 團隊介紹凸顯相關經驗
- [ ] 時程經費合理可行

### 簡報準備檢核

- [ ] 投影片數量適中（12-15 頁）
- [ ] 字體夠大、圖表清楚
- [ ] 練習過多次
- [ ] 準備補充資料
- [ ] 預想可能問題
