# SBIR 計畫書數據呈現技巧

> 「沒有數據的觀點只是意見，有數據支持的觀點才是洞見。」

## 數據的說服力

### 為什麼數據重要？

評審委員在審查計畫書時，會特別關注：

| 評審關注點 | 需要的數據 |
|------------|------------|
| 問題是否真實存在？ | 產業統計、市場調查 |
| 技術是否可行？ | 實驗數據、前期成果 |
| 市場是否夠大？ | 市場規模、成長率 |
| 效益是否合理？ | 量化目標、ROI 計算 |
| 團隊是否有能力？ | 過往績效、相關經驗 |

### 量化 vs 質化描述

**❌ 質化描述（無說服力）**
> 「本技術可大幅提升生產效率，降低成本，提高產品品質。」

**✅ 量化描述（有說服力）**
> 「根據前期實測，本技術可：
> - 提升生產效率 35%
> - 降低製造成本 22%
> - 將不良率從 3.5% 降至 0.8%」

## 圖表設計原則

### 何時該用圖表？

| 資料類型 | 建議圖表 |
|----------|----------|
| 趨勢變化 | 折線圖 |
| 比較大小 | 長條圖 |
| 佔比分布 | 圓餅圖 |
| 流程關係 | 流程圖 |
| 多維比較 | 雷達圖 |
| 時間規劃 | 甘特圖 |

### 圖表設計 5 原則

#### 1. 簡潔明瞭
- 一張圖表只傳達一個重點
- 移除不必要的裝飾元素
- 使用清晰的標題

#### 2. 數據準確
- 起始軸從 0 開始（除非有特殊理由）
- 避免扭曲比例
- 標註數據來源

#### 3. 視覺突出
- 使用顏色突顯重點
- 關鍵數據加粗或放大
- 使用對比色區分

#### 4. 易於閱讀
- 字體大小適中
- 圖例位置清楚
- 必要時加上說明文字

#### 5. 一目瞭然
- 5 秒內能理解重點
- 不需要長篇解釋
- 支持論述主軸

**❌ 問題圖表**
- 太多數據線

## 量化目標設定

### SMART 原則

| 要素 | 說明 | 範例 |
|------|------|------|
| **S**pecific | 具體明確 | 導入 AI 品質檢測系統 |
| **M**easurable | 可量測 | 不良率降低至 1% 以下 |
| **A**chievable | 可達成 | 基於前期實測數據 |
| **R**elevant | 相關性 | 與計畫目標直接相關 |
| **T**ime-bound | 有時限 | 12 個月內達成 |

### 查核點量化範例

#### Phase 1（6 個月）

| 季度 | 查核項目 | 量化指標 |
|------|----------|----------|
| Q1 | 完成需求分析 | 訪談 10 家潛在客戶 |
| Q1 | 完成技術規格 | 產出技術規格書 1 份 |
| Q2 | 完成雛型開發 | 雛型系統 1 套 |
| Q2 | 完成功能驗證 | 準確率達 85% 以上 |

#### Phase 2（18 個月）

| 階段 | 查核項目 | 量化指標 |
|------|----------|----------|
| M1-M6 | 核心模組開發 | 完成 3 個核心模組 |
| M7-M12 | 系統整合測試 | 通過 50 項測試案例 |
| M13-M18 | 實場驗證 | 導入 3 家示範客戶 |

## 效益量化範例

### 技術效益

```
本計畫技術效益量化如下：

1. 生產效率
   - 現況：人工檢測每小時 100 件
   - 目標：AI 檢測每小時 1,000 件
   - 提升：10 倍

2. 檢測準確率
   - 現況：人工檢測 92%
   - 目標：AI 檢測 99.5%
   - 提升：7.5 個百分點

3. 人力需求
   - 現況：每班 5 名檢測員
   - 目標：每班 1 名監控員
   - 節省：80% 人力
```

### 財務效益

```
預期財務效益（以單一客戶導入為例）：

投入成本：
- 系統導入費    300 萬元
- 年維護費       30 萬元

年度效益：
- 人力節省      240 萬元（4人 × 60萬/年）
- 不良品減少    150 萬元
- 產能提升       80 萬元
  ─────────────────────
  年度總效益    470 萬元

投資回收期：約 8 個月
三年 ROI：370%
```

### 市場效益

```
TAM/SAM/SOM 分析：

TAM（總潛在市場）
台灣製造業品質檢測市場
= 製造業家數 × 平均導入費用
= 150,000 家 × 100 萬
= 1,500 億元/年

SAM（可觸及市場）
具規模且有數位化需求的製造業
= TAM × 20%
= 300 億元/年

SOM（可獲得市場）
計畫完成後 3 年內目標市佔
= SAM × 1%
= 3 億元/年
```

## 常見錯誤與修正

### 錯誤 1：數據來源不明

**❌ 問題**
> 「根據統計，市場規模達 500 億元。」

**✅ 修正**
> 「根據工研院 IEK 2024 年報告，台灣智慧製造市場規模達 500 億元（來源：工研院產業情報暨知識服務平台）。」

### 錯誤 2：數據過時

**❌ 問題**
> 「根據 2018 年調查...」

**✅ 修正**
- 使用最新數據（2-3 年內）
- 若無最新數據，說明並推估

### 錯誤 3：目標不切實際

**❌ 問題**
> 「計畫完成後第一年營收 10 億元」（對新創而言）

**✅ 修正**
- 基於實際市場規模和產能推估
- 分階段設定目標
- 說明推算邏輯

### 錯誤 4：缺乏佐證

**❌ 問題**
> 「本技術準確率可達 99%」（無實驗數據）

**✅ 修正**
> 「經過 3 個月、1,000 筆測試資料的驗證，本技術準確率達 98.7%，詳見附件之測試報告。」

## 數據呈現自我檢核

- [ ] 每個關鍵論點都有數據支持？
- [ ] 數據來源都有標註？
- [ ] 數據是最新的（3 年內）？
- [ ] 目標設定有推算邏輯？
- [ ] 圖表簡潔易懂？
- [ ] 重要數據有視覺突顯？
- [ ] 效益計算合理可信？
