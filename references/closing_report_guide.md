# 結案報告撰寫指南

> 結案報告是計畫的成績單，寫好它，為未來申請加分。

## 結案報告的重要性

### 為什麼要認真寫結案報告？

1. **影響結案審查**：報告品質影響結案通過
2. **累積信譽**：良好紀錄有助未來申請
3. **成果記錄**：日後提案可引用
4. **智財基礎**：專利申請的參考

---

## 結案報告結構

### 標準結構

```
結案報告目錄

1. 計畫摘要
   1.1 計畫目標
   1.2 主要成果
   1.3 效益總結

2. 計畫執行內容
   2.1 執行概況
   2.2 工作項目與查核點達成情形
   2.3 技術內容說明

3. 研發成果
   3.1 技術成果
   3.2 產品/雛型
   3.3 專利/論文
   3.4 其他成果

4. 效益評估
   4.1 技術效益
   4.2 經濟效益
   4.3 社會效益

5. 經費執行
   5.1 經費執行情形
   5.2 各項經費明細

6. 後續規劃
   6.1 商品化規劃
   6.2 市場推廣
   6.3 後續研發

7. 檢討與建議

附件
- 測試報告
- 專利證書
- 合作證明
- 其他佐證
```

## 各章節撰寫要點

### 第一章：計畫摘要

**目的**：讓審查者 2 分鐘內掌握全貌

**內容**：
- 計畫目標（1-2 句）
- 主要成果（3-5 點）
- 量化效益（關鍵數據）

**範例**：
```
1.1 計畫目標

本計畫開發應用於中小型製造業的 AI 預測維護系統，
目標在 12 個月內完成系統開發、取得 3 家合作工廠驗證。

1.2 主要成果

1. 完成 AI 預測維護平台開發，預測準確率達 93%
2. 取得 5 家（超越原目標 3 家）工廠實證驗證
3. 申請發明專利 2 件（1 件已核准）
4. 發表技術論文 1 篇（MDPI Sensors）

1.3 效益總結

└─ 技術效益：較現有方案準確率提升 35%
└─ 經濟效益：預估每年可為客戶節省 300 萬維護成本
└─ 就業效益：計畫期間新聘研發人員 3 人
```

│ Q1-2      │ 完成架構設計  │ 如期完成      │ 100%│
│ Q2-1      │ 完成核心模組  │ 如期完成      │ 100%│

### 第三章：研發成果

**目的**：具體展示計畫產出

**技術成果呈現**：

```
3.1 技術成果

3.1.1 核心技術突破

本計畫完成之核心技術包括：

1. 多感測器融合演算法
   - 整合振動、溫度、電流等 8 種訊號
   - 自動特徵擷取與權重調整
   - 較單一訊號方法準確率提升 40%

2. 輕量化邊緣 AI 模型
   - 模型大小僅 50MB
   - 可運行於工業電腦
   - 推論速度 < 100ms

3. 自適應學習機制
   - 可依產線特性自動調參
   - 減少人工調校時間 80%

技術比較：

┌──────────────────────────────────────────┐
│ 指標        │ 計畫技術 │ 現有技術 │ 提升 │
├─────────────┼──────────┼──────────┼──────┤
│ 預測準確率  │   93%    │   65%    │ +43% │
│ 提前預警    │  72 小時 │  24 小時 │  3x  │
│ 誤報率      │   5%     │   20%    │ -75% │
└──────────────────────────────────────────┘
```

**產品/雛型**：
- 附上照片或截圖
- 說明功能與規格
- 展示使用情境

**專利/論文**：
- 列出申請/核准狀態
- 附上證明文件

### 第四章：效益評估

**目的**：量化說明計畫價值

**範例**：

```
4.1 技術效益

1. 技術指標達成
   □ 預測準確率：目標 90%，達成 93%
   □ 適用設備：目標 3 類，達成 5 類
   □ 專利申請：目標 1 件，達成 2 件

2. 技術水準提升
   □ 較計畫前提升：準確率 +35%
   □ 較國內水準：領先同業約 2 年

4.2 經濟效益

1. 直接效益（計畫執行期間）
   └─ 新增營收：150 萬元（試銷收入）
   └─ 成本節省：研發人力培訓節省 50 萬

2. 預期效益（未來 3 年）
   └─ 預估營收：3,000 萬元
   └─ 預估毛利：1,200 萬元（毛利率 40%）

3. 客戶效益
   └─ 每家客戶年省維護成本：300 萬元
   └─ 投資回收期：約 6 個月

4.3 社會效益

1. 就業增加
   └─ 計畫期間新聘：3 人
   └─ 預計 3 年內增聘：10 人

2. 人才培育
   └─ 培訓 AI 研發人員：5 人
   └─ 發表技術論文：1 篇

3. 產業貢獻
   └─ 帶動設備商合作：2 家
   └─ 協助傳產升級：5 家工廠
```

## 常見缺失與避免

| 缺失 | 問題 | 避免方法 |
|------|------|----------|
| 內容空泛 | 缺乏具體數據 | 全部量化 |
| 查核點未達 | 沒有說明原因 | 誠實說明並提對策 |
| 經費差異 | 未說明流用原因 | 詳細說明 |
| 缺少佐證 | 沒有附件證明 | 完整附件 |
| 效益誇大 | 不合理估算 | 保守合理 |

## 結案簡報重點

### 簡報結構（20 分鐘）

| 段落 | 時間 | 內容 |
|------|------|------|
| 計畫回顧 | 2 min | 目標提醒 |
| 執行成果 | 5 min | 查核點達成 |
| 技術亮點 | 5 min | 核心突破 |
| 效益展示 | 3 min | 量化效益 |
| Demo | 3 min | 展示成果 |
| 後續規劃 | 2 min | 商品化計畫 |

### 簡報技巧

- 準備實際 Demo 或影片
- 圖表多於文字
- 突顯超越預期的成果
- 誠實面對不足
