# 聯合申請指南

> 兩家以上企業聯合申請，補助上限加倍，但複雜度也加倍。

## 聯合申請概述

### 何謂聯合申請？

**定義**：兩家以上符合資格的中小企業，共同提出 SBIR 計畫申請

**補助上限**：

| 申請類型 | Phase 1 | Phase 2 |
|----------|---------|---------|
| 個別申請 | 150 萬 | 1,200 萬 |
| 聯合申請 | 300 萬 | 3,000 萬 |

**每家上限**：仍為個別上限

---

## 何時適合聯合申請？

### ✅ 適合聯合申請的情況

| 情境 | 說明 |
|------|------|
| 技術互補 | A 有硬體，B 有軟體 |
| 產業鏈合作 | 上中下游整合 |
| 資源互補 | A 有技術，B 有場域 |
| 規模效益 | 單獨做不夠規模 |
| 市場互補 | 不同客群，相同產品 |

### ❌ 不適合聯合申請的情況

| 情境 | 說明 |
|------|------|
| 為補助拼湊 | 沒有實質合作基礎 |
| 分工不明 | 責任歸屬不清 |
| 利益衝突 | 潛在競爭關係 |
| 管理困難 | 溝通協調成本太高 |

## 聯合申請案例

### 成功案例

**案例：智慧農業解決方案**

```
合作夥伴：
├─ A 公司（主導）：感測器硬體製造商
├─ B 公司：農業 AI 軟體新創
└─ C 公司：農業設備代理商

分工：
├─ A：開發農業環境感測器
├─ B：開發 AI 預測分析系統
└─ C：場域驗證與市場推廣

成果：
├─ 整合型智慧農業平台
├─ 在 10 個農場實測
└─ 產品一年內上市
```

**成功因素**：
- 技術高度互補
- 分工明確
- 有既有合作經驗
- 共同客戶群

---

## 合作夥伴選擇

### 評估標準

| 標準 | 權重 | 評估方法 |
|------|------|----------|
| 技術互補性 | 高 | 技術能力對照 |
| 資源互補性 | 高 | 場域、設備、人才 |
| 合作意願 | 高 | 高層承諾 |
| 信任基礎 | 中 | 過往合作經驗 |
| 財務穩定 | 中 | 財務報表 |
| 文化契合 | 中 | 工作風格 |

### 夥伴來源

1. **產業鏈關係**：客戶、供應商
2. **產業公會**：同業交流
3. **研發聯盟**：科專計畫認識
4. **產學合作**：大學技轉對象

## 分工與權責

### 分工規劃

```
計畫組織架構

        ┌─────────────────┐
        │   計畫主持人    │
        │  （主導廠商）   │
        └────────┬────────┘
                 │
    ┌────────────┼────────────┐
    │            │            │
┌───┴───┐  ┌─────┴─────┐  ┌───┴───┐
│ A 公司 │  │ B 公司    │  │ C 公司 │
│(主導)  │  │(協助)     │  │(協助)  │
├────────┤  ├───────────┤  ├────────┤
│WP1:硬體│  │WP2:軟體   │  │WP3:驗證│
│budget: │  │budget:    │  │budget: │
│500萬   │  │300萬      │  │200萬   │
└────────┘  └───────────┘  └────────┘
```

### 必須約定的事項

| 事項 | 說明 |
|------|------|
| 計畫主持人 | 誰是總負責人？ |
| 工作分配 | 各自負責什麼？ |
| 經費分配 | 各自多少預算？ |
| 智財歸屬 | 成果歸誰？ |
| 商業化權利 | 誰可以商業化？ |
| 爭議處理 | 如何解決爭議？ |

## 經費分配

### 分配原則

1. **依工作量分配**：按實際投入
2. **依貢獻度分配**：按技術貢獻
3. **依資源投入分配**：按場域設備

### 分配範例

**1,000 萬聯合申請案**：

| 公司 | 角色 | 工作比例 | 經費 |
|------|------|----------|------|
| A（主導）| 硬體 | 50% | 500 萬 |
| B | 軟體 | 30% | 300 萬 |
| C | 驗證 | 20% | 200 萬 |

**注意**：每家不超過個別申請上限

## 智財處理

### 智財歸屬模式

| 模式 | 說明 | 適用情況 |
|------|------|----------|
| 獨立歸屬 | 各自成果各自擁有 | 分工明確 |
| 共同擁有 | 共同開發共同擁有 | 整合成果 |
| 歸主導者 | 成果歸主導廠商 | B/C 公司同意 |
| 依貢獻分配 | 按貢獻比例擁有 | 公平分配 |

### 合約必備條款

```
智財權條款範例

第 X 條 智慧財產權歸屬

1. 前景智財（計畫產出）
   (1) 各方獨立完成之成果，歸該方所有
   (2) 共同完成之成果，由雙方共同擁有，持分比例為 [     ]
   (3) 改良發明依 [     ] 原則處理

2. 背景智財（既有技術）
   (1) 各方既有技術仍歸各方所有
   (2) 計畫執行期間，各方授權他方使用
   (3) 計畫結束後，授權 [繼續有效 / 終止]

3. 商業化權利
   (1) [主導廠商 / 各方] 有權將成果商業化
   (2) 授權金分配比例為 [     ]
   (3) 經 [書面同意 / 不需同意] 得轉授權
```

## 計畫書撰寫要點

### 必須呈現的內容

1. **合作必要性**
   - 為什麼要合作？
   - 各自能力為何？
   - 單獨做行不行？

2. **分工規劃**
   - 各自負責什麼？
   - 工作如何銜接？
   - 如何確保整合？

3. **管理機制**
   - 如何協調溝通？
   - 如何解決爭議？
   - 查核點如何設定？

### 範例寫法

```
1.3 聯合申請必要性

本計畫採聯合申請之必要性分析如下：

1. 技術互補性
   ┌───────────────────────────────────────┐
   │ 能力      │ A 公司  │ B 公司  │ 整合  │
   ├───────────┼─────────┼─────────┼───────┤
   │ 硬體設計  │   ★★★   │    ★    │  ✓    │
   │ 軟體開發  │    ★    │   ★★★   │  ✓    │
   │ 系統整合  │   ★★    │   ★★    │  ✓    │
   └───────────────────────────────────────┘

2. 資源互補性
   - A 公司：生產設備、品質認證
   - B 公司：AI 技術、雲端平台

3. 單獨執行之困難
   - A 公司：缺乏軟體開發能力
   - B 公司：缺乏硬體製造能力
   - 結論：聯合申請為唯一可行方案
```

## 常見問題

### Q1：聯合申請通過率較高嗎？
**A**：不一定。聯合申請複雜度高，如果分工不清或整合不佳，反而扣分。

### Q2：可以事後增加合作夥伴嗎？
**A**：原則上不行。如有特殊需求需向計畫辦公室申請。

### Q3：如果合作夥伴中途退出怎麼辦？
**A**：需立即通報並提出因應方案，可能影響計畫繼續執行。

### Q4：兩家公司可以申請同一領域嗎？
**A**：可以，但需說明分工差異和整合價值。

## 聯合申請檢核清單

### 申請前

- [ ] 確認合作夥伴符合 SBIR 資格
- [ ] 確認技術/資源互補性
- [ ] 確認高層支持與承諾
- [ ] 完成分工與經費分配討論
- [ ] 簽署合作意向書或合約

### 撰寫時

- [ ] 說明聯合申請必要性
- [ ] 呈現各家能力與分工
- [ ] 明確經費分配表
- [ ] 規劃管理協調機制
- [ ] 處理智財歸屬

### 執行時

- [ ] 建立定期溝通機制
- [ ] 各自追蹤進度
- [ ] 定期彙整報告
- [ ] 處理突發狀況
